Security

Six layers. Zero trust.

Every tool call passes through 6 security layers before execution. No exceptions.

01

Policy Enforcement

Per-tool allow/deny/ask rules. Rate limiting. Argument validation. Your rules, deterministically enforced.

02

Runtime Monitoring

Dynamic call-chain tracking via DAG. Anomaly detection. Circuit breaker for fault isolation.

03

Transport Proxy

Transparent MCP proxy — intercepts stdio/SSE without modifying the server. Zero server-side changes.

04

Trust Verification

TOFU (Trust On First Use) + Ed25519 certificates. Cryptographic server identity verification.

05

Security Scanner

Static analysis of MCP server capabilities. Risk scoring. Dangerous pattern detection.

06

Auto Installer

Auto-detects and injects into all MCP client configs. Claude, Cursor, Windsurf, Cline, Continue, Zed.

Security Audit Report

Run jadegate scan --audit to get a detailed CT-style security report for all your MCP servers.

Trust Model

TOFU + Ed25519 Trust

Trust On First Use — like SSH known_hosts. First connection fingerprints the server. Subsequent connections verify identity. Ed25519 certificates for cryptographic proof.

🔑 TOFU — Trust On First Use
│
├── 💠 First connection — fingerprint stored in local trust store
│
├── 🔒 Subsequent calls — Ed25519 signature verified against stored fingerprint
│ ├── ✅ Match → proceed
│ └── ❌ Mismatch → circuit breaker triggered
│
└── 📋 jadegate scan — audit all servers, review trust decisions

Certificate Management

Every tool gets a security certificate with risk profile, trust score, and cryptographic signature. View all with jadegate cert list.
